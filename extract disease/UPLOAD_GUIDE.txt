【1】 귤(과일) 이미지 — 직접 업로드 (필수)
--------------------------------------------------------------------------------
  올릴 위치 (이 폴더 기준):

    uploads/Tangerine_2D/<클래스폴더이름>/<이미지파일들>

  규칙:
  - torchvision ImageFolder 와 동일합니다.
  - 반드시 "클래스마다 하위 폴더"가 있어야 합니다. (클래스 = 병해 이름 등 라벨)
  - 한 클래스에 여러 장의 이미지를 넣을 수 있습니다.

  예시 구조:

    uploads/Tangerine_2D/
      Black_spot/
        img_001.jpg
        img_002.png
      Canker/
        a.jpg
      Healthy/
        h01.jpeg

  지원 확장자: .png .jpg .jpeg .webp .bmp .tif .tiff

  주의:
  - 이미지를 uploads/Tangerine_2D 바로 아래에 두면 안 됩니다. 반드시 클래스 폴더 안에.
  - 빈 폴더만 있고 이미지가 없으면 해당 클래스는 건너뜁니다.


【2】 SAM 가중치 파일 (선택 — 없으면 휴리스틱 폴백만 사용)
--------------------------------------------------------------------------------
  올릴 위치:

    checkpoints/sam_vit_b_01ec64.pth

  또는 decal_colab.yaml 의 decal.sam_checkpoint 를 절대 경로로 수정.

  공식 ViT-B 다운로드 URL (노트북에서 wget 으로 받을 수 있음):
    https://dl.fbaipublicfiles.com/segment_anything/sam_vit_b_01ec64.pth

  파일이 없으면:
  - rembg(배경 제거) + LAB 명도 기반 병변 후보만 사용합니다.
  - SAM보다 마스크 품질이 떨어질 수 있습니다.


【3】 Colab 에서 이 폴더를 쓰는 방법
--------------------------------------------------------------------------------
  A) Google Drive 에 압축(zip)으로 올린 뒤 Colab 에서 압축 해제
  B) 왼쪽 파일 패널에서 uploads/Tangerine_2D 아래로 폴더·이미지 드래그 업로드
  C) 전체 Tangerine 레포를 클론한 경우: 레포 안의 Colab_From2D 로 이동해 실행


【4】 산출물 (실행 후 자동 생성 — 업로드할 필요 없음)
--------------------------------------------------------------------------------
    output/decal_cache/<클래스명>/*.png   … 병변 RGBA 패치
    output/decal_cache/manifest.json     … 이미지별 패치 목록


【5】 이 폴더에 이미 들어 있는 것 (업로드 불필요)
--------------------------------------------------------------------------------
    decal_colab.yaml       … 경로·SAM 옵션
    run_decal_cache.py     … 실행 스크립트
    mask_and_patch.py      … 전처리 로직 (레포 src 와 동기화 권장)
    requirements-colab.txt … pip 의존성
