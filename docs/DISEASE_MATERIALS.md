# 병해 클래스별 절차 재질 스펙 (읽는 사람용)

감귤 표면은 **원래 알베도(색)에 살짝 틴트를 입힌 뒤**, 그 위에 수학 패턴으로 병만 덧칠하는 방식이다.  
설정 파일은 `data/Tangerine_3D/configs/variants_batch.yaml` 안의 **`disease_params`** 블록이고, 실제 그림은 `src/blender_sim/disease_overlays.py` 가 만든다.

아래에서 **색 코드**는 `#` 육 자리(웹에서 흔히 쓰는 형식)이며, **옆에 어떤 색인지** 한글로 적어 두었다.

---

## 설정 항목을 읽는 법

YAML 에 적는 이름은 영문이라 헷갈릴 수 있다. 이 문서에서는 각 항목마다:

- **무엇을 바꾸는지** — 한글로 먼저 설명
- **파일에서 쓰는 이름** — 괄호 안에 적음 (복사해 넣을 때용)

숫자는 “클수록 어떻게 되는지” 위주로 적었다.

## Black spot — 검은점무늬병

### 어떻게 보이게 하려는지

껍질에 **작은 검은 점**이 여러 개 박힌 것처럼. 점 안은 거의 검정에 가깝고, 주변보다 **더 거칠게** 보이며, 아주 약한 **요철**이 있다. 표면을 깊게 파이지는 않는다.

### 패턴이 붙는 방식

과일 메쉬와 같이 움직이는 **물체 좌표**를 쓴다. 그래서 껍질을 따라 점이 도망가지 않고 붙어 있는 느낌이 난다.

### 안쪽에서 하는 일 (개념만)

1. **점무늬 격자** 같은 수학 패턴(Voronoi)으로 “여기가 점 중심, 여기는 가장자리” 거리를 만든다.
2. 그 거리를 0~1 로 맞춘 뒤, **그라데이션**으로 “점 안은 어둡고 밖은 밝다”로 만든다.
3. 그 결과를 **반점 마스크**라 부르자: 어두운 쪽만 병 색이 보인다.
4. 최종 색 = (건강하게 보이는 베이스)와 (아주 어두운 반점 색)을 그 마스크로 섞은 것.

### 쓰는 색 (코드 옆 = 느낌)

| 코드 | 어떤 색인지 |
|------|-------------|
| `#0A0605` | 거의 검정에 가까운 아주 어두운 갈색 (그라데이션 쪽) |
| `#050302` | 반점에 최종으로 얹는 **더 진한** 거의 검정 |

### 표면 물성

- **거칠기**: 건강 부분은 원래 거칠기, 점 안은 더 매트하게(숫자로 0.85 근처까지 올림).
- **요철**: 아주 미세한 노이즈로 범프만 준다. **깊이 변형(디스플레이스)은 쓰지 않는다.**

### `variants_batch.yaml` 에서 건드릴 수 있는 것

| 먼저 읽을 설명 | 파일 안 이름 (영문) | 대략 이렇게 동작 |
|----------------|---------------------|------------------|
| 점이 **촘촘한지 듬성한지** | `voronoi_scale` | 숫자를 크게 하면 점이 작아지고 많아 보인다. |
| 점 배치가 **규칙적인지 삐뚤한지** | `voronoi_randomness` | 높이면 덜 “격자” 같아질 수 있다. |
| (Blender 4.x) 점 경계가 **부드러운지** | `voronoi_smooth` | 크면 경계가 덜 날선 느낌. |
| 반점 **덩어리 크기** 감 | `spot_dist_map_max` | 거리를 잘라 넣는 상한. 옆으로 퍼 보이는 정도에 영향. |
| 어디부터 **어둡게 칠할지** (그라데이션 슬라이더 위치) | `spot_ramp_pos` | 작으면 더 좁은 점, 크면 넓어질 수 있음. |
| 경계가 **계단처럼 보일지 부드러울지** | `spot_ramp_interpolation` | 보통 `LINEAR` 가 자연스럽다. |
| 반점이 **더 진하게/흐리게** 보이게 | `spot_mix_boost` | 대비를 키운다. |
| 그라데이션 끝 색 (어두운 쪽) | `spot_color_hex` | 위 표 `#0A0605` 류. |
| 최종으로 섞는 반점 색 | `spot_dark_hex` | 위 표 `#050302` 류. |
| 점 위만 **얼마나 매트하게** | `spot_rough` | 0~1, 높을수록 더 거칠게. |
| 점 위 **요철 세기** | `overlay_bump_strength` | 너무 크면 인위적으로 튀어 보일 수 있음. |

## Canker — 궤양병

### 어떻게 보이게 하려는지

**덩어리 난 병반**: 가운데는 코르크·나무껍질 같은 갈색, 바깥으로 갈수록 물이 번진 듯한 **연한 노랑 테두리**, 그 밖은 과일 원래 색에 가깝게. 실제로 메쉬가 **살짝 움푹 들어가거나** 가장자리가 **살짝 올라온** 느낌까지 준다.

### 쓰는 색 (코드 옆 = 느낌)

| 코드 | 어떤 색인지 |
|------|-------------|
| `#6B4E31` | 궤양 **중심** — 적갈색·코르크 느낌 |
| `#F9E076` | **물침(할로)** — 밝은 레몬·크림 노랑 |
| `#FF9E1B` | 병밖으로 나가면서 **과일 살색**에 가까운 오렌지 (코드에 고정) |

이 세 가지는 지금은 **프로그램 안에 박혀 있어서**, YAML 로만 바꾸기 어렵고 색을 바꾸려면 `disease_overlays.py` 의 궤양 부분을 직접 수정해야 한다.

### 안쪽에서 하는 일 (개념만)

1. “**가장자리까지의 거리**” 패턴으로 덩어리 모양을 만든다 (점무늬와 다른 종류의 Voronoi).
2. 그 거리를 0~1 로 맞춘 값으로, 위 세 색을 **한 줄 그라데이션**처럼 이어 붙여 궤양 단면 색을 만든다.
3. 또 다른 구간 나눔으로 “**얼마나 넓게 병으로 덮을지**”를 정한다 → 그걸로 건강 색과 섞는 강도가 정해진다.
4. **거칠기**도 같은 거리 정보로 구역마다 다르게 준다.
5. **깊이**는 그 거리를 높이 맵으로 바꿔서, 중심은 조금 파이고 가장자리는 조금 튀게(대략 −0.002 ~ +0.003 스케일) 디스플레이스에 넣는다. 전체 세기는 아래 `disp_scale` 로 조절.

### `variants_batch.yaml` 에서 건드릴 수 있는 것

| 먼저 읽을 설명 | 파일 안 이름 (영문) | 대략 이렇게 동작 |
|----------------|---------------------|------------------|
| 덩어리 **크기·개수** 감 | `lesion_scale` | 숫자 의미는 “패턴 확대/축소”에 가깝다. |
| 첫 거리 맵이 **얼마나 넓게 퍼질지** | `canker_de_span` | 궤양 윤곽이 잡히는 범위. |
| 병으로 **덮을 거리 구간 시작·끝** | `canker_overlay_in`, `canker_overlay_out` | 이 사이만 병 색이 강하게 섞인다. |
| 전체적으로 **병 색이 얼마나 섞일지** | `overlay_strength` | 1에 가까울수록 원래 과일색이 덜 보인다. |
| **움푹·볼록 세기** 전체 배율 | `disp_scale` | 1이 기본; 크면 형이 더 과해질 수 있다. |

## Greening — 녹화 (HLB 느낌)

### 어떻게 보이게 하려는지

**넓게 번진 얼룩**으로, 황록·올리브·진한 녹이 섞여 **색이 바래고 녹스는** 느낌. 입체 요철은 거의 두지 않고 **색만**으로 표현한다.

### 쓰는 색 (예시 코드 옆 = 느낌)

| 코드 | 어떤 색인지 |
|------|-------------|
| `#E88A28` | 살아 있는 귤 살색에 가까운 오렌지 |
| `#A4C060` | 황록 — 노랑과 녹의 중간 |
| `#4A7D3E` | 올리브 녹색 |
| `#2F6A38` (기본 예시) | 진한 잎 녹색 — **녹 기운**을 한 번 더 얹을 때 |

YAML 에서 `greening_yellow_hex` 등으로 바꿀 수 있다.

### 안쪽에서 하는 일 (개념만)

1. **큰 덩어리 노이즈** 두 개를 섞어 **얼룩 모양**을 만든다.
2. 과일 **아래쪽**(세로 축)일수록 녹이 더 들어가게 **가중치**를 곱한다 — 아래가 더 “녹스는” 느낌.
3. 위 색들을 단계적으로 섞어 최종 **녹화 색**을 만든 뒤, 건강 베이스와 섞는다.

### `variants_batch.yaml` 에서 건드릴 수 있는 것

| 먼저 읽을 설명 | 파일 안 이름 (영문) | 대략 이렇게 동작 |
|----------------|---------------------|------------------|
| 큰 얼룩의 **크기** | `mottle_scale` | 작으면 얼룩이 잘게, 크면 크게. |
| 보조 패턴 **세밀함** | `greening_noise_scale` | 질감이 더 잘게 갈라 보일 수 있음. |
| 얼룩이 **어디부터 진하게 보일지** (명도 구간) | `mottle_ramp_lo`, `mottle_ramp_hi` | 두 값 사이에서 녹이 강해진다. |
| 전체 **녹이 얼마나 섞일지** | `overlay_strength` | |
| 녹을 **한 번 더 강조** | `greening_boost` | |
| 표면 **전체 거칠기** (한 값으로 고정) | `roughness` | 병만 따로 거칠게 하는 게 아니라 통째로. |
| 단계별 색 (위 표) | `greening_orange_hex`, `greening_yellow_hex`, `greening_olive_hex`, `green_tint_hex` | 원하는 톤으로 맞추기. |

“아래쪽만 더 녹게” 하는 범위는 코드 안 숫자라서, 그걸 바꾸려면 `overlay_greening` 부분을 수정한다.

## Scab — 더덩이병

### 어떻게 보이게 하려는지

**울퉁불퉁한 딱지**가 붙은 듯한 구역. 색은 **회끼 도는 갈색**, 표면은 **거의 최대로 무광**에 가깝게, 메쉬가 **살짝 솟아** 보이게 한다.

### 쓰는 색

| 코드 | 어떤 색인지 |
|------|-------------|
| `#8D7B68` | 탁한 회갈색·딱지색 (프로그램에 고정, YAML 로는 안 빠짐) |

바꾸려면 `disease_overlays.py` 의 scab 부분을 직접 고친다.

### 안쪽에서 하는 일 (개념만)

1. **서로 다른 두 종류의 거친 노이즈**를 섞어 **불규칙한 덩어리 마스크**를 만든다.
2. 그 마스크로 건강 색과 딱지 색을 섞고, 같은 마스크로 **거칠기**와 **살짝 튀어 오른 높이**에도 쓴다.

### `variants_batch.yaml` 에서 건드릴 수 있는 것

| 먼저 읽을 설명 | 파일 안 이름 (영문) | 대략 이렇게 동작 |
|----------------|---------------------|------------------|
| 첫 번째 질감 패턴 **크기** | `scab_noise_a_scale` | |
| 두 번째(살이 더 찢어진 느낌) **크기** | `scab_noise_b_scale` | |
| 딱지가 **넓게 잡힐지 좁게 잡힐지** (그라데이션 기준) | `scab_mask_pos` | 숫자에 따라 건강/병 경계가 움직인다. |
| **튀어 오르는 높이** | `scab_disp_scale` | 크면 딱지가 더 도드라져 보임. |
| 색·거칠기·높이에 공통으로 걸리는 **세기** | `overlay_strength` | |

## 품질을 더 올리고 싶을 때

### 렌더로 굽는 해상도

`variants_batch.yaml` 의 **`gltf_bake_size`** (기본 2048): 숫자를 키우면 **무늬 경계가 덜 깨져** 보인다. 대신 시간·메모리가 늘어난다.

### 위 표만으로 부족할 때

- **실제 사진에 더 가깝게** 하려면: 나중에 **거칠기 맵·노멀 맵**을 그림으로 만들어 붙이거나, Substance 같은 툴을 쓰는 방법이 있다 (작업량은 큼).
- **반점만 격자 같음**: 위 Black spot 표에서 “촘촘함”, “경계 부드러움”, “덩어리 크기” 줄을 순서대로 만져 본다.
- **궤양이 너무 큼/작음**: Canker 표의 덩어리 크기·덮는 구간·섞기 강도.
- **녹이 약함/과함**: Greening 표의 섞기·부스트·얼룩 구간.

### 참고용으로 만든 2D 그림

`scripts/gen_disease_texture_masks.py` 가 만든 PNG 는 **미리 보기·비교용**에 가깝다. 지금 메인으로 쓰는 건 위 **절차 셰이더 → 굽기** 경로다.
